## Improvement candidates (prioritized)

### 1. Unauthenticated `POST /api/internal/ban` — **Critical**

Exposed ban write with no auth. Lock with `X-Admin-Key` or remove if dead code. Add test for 401 without key.

### 2. Unhandled `TokenRateLimitError` in bot handlers — **High**

`on_join_request` / `start_command` crash when user exceeds token quota; user left restricted without DM. Catch and notify gracefully.

### 3. Stylometry matching loop may attribute wrong `matched_ban` — **High**

`check_known_bad` stylometry block reloads all profiles per ban; add test + fix loop to compare against correct ban profile.

### 4. Rate limit bypass via spoofed `X-Forwarded-For` — **Medium**

`_client_ip` trusts client header. Add `TRUSTED_PROXY` config; test submit 429; unit-test limiter window.

### 5. Watcher re-alerts same match every interval — **Medium**

`run_watcher_job` has no deduplication; log channel spam. Track alerted pairs or debounce.

### 6. Admin API: timing-safe compare + rate limit — **Medium**

`!=` on API key; no brute-force throttle. Use `secrets.compare_digest`; limit `/api/admin/*`.

### 7. `channel_id` mismatch `/verify` vs join-request — **Medium**

`start_command` uses `settings.channel_id`; join uses `request.chat.id`. Document or unify.

### 8. Expand bot handler tests — **Medium** (req 3)

`apply_verification_decision`, `on_join_request`, `on_callback` with mocks per telegram-bot-architect patterns.
